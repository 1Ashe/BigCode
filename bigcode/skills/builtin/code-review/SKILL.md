# Code Review

Use this skill when asked to review a patch, branch, pull request, or local changes.

Workflow:

1. Prioritize correctness, regressions, security, data loss, and missing tests.
2. Ground every finding in a specific file and line when possible.
3. Lead with findings ordered by severity.
4. Keep summaries secondary and concise.
